# 如何發送郵件通知

本系統提供兩種手動發送郵件通知的方式，你可以根據需求選擇最適合的方法。

> **重要說明**：這是**手動發送**功能，需要明確指定要發送的文章 slug。系統**不會**自動檢測新文章並發送郵件。每次發送都需要執行命令或調用 API。

## 前置要求

1. **設定環境變數** `BUTTONDOWN_API_KEY`
   - 在 `.env.local` 檔案中設定你的 Buttondown API Key
   - 如果使用 Vercel 部署，請在 Vercel 專案設定中添加環境變數

2. **確保文章已經部署上線**
   - 文章必須已經推送到 Git 並完成 Vercel 部署
   - 確認文章可以在網站上正常訪問

## 方法一：命令行腳本（推薦）

### 發送指定文章的通知

```bash
npm run newsletter:send -- --slug "2026/01/文章文件名"
```

### 完整示例

```bash
# 發送最新文章
npm run newsletter:send -- --slug "2026/01/旅行的意義-大二寒假環西半部"

# 注意：slug 格式為 YYYY/MM/文件名（不含 .md 副檔名）
```

### 自訂郵件標題和內容

```bash
npm run newsletter:send -- --subject "自訂標題" --body "自訂內容"
```

### 混合使用（指定文章但自訂標題）

```bash
npm run newsletter:send -- --slug "2026/01/文章文件名" --subject "自訂標題"
```

### 強制重新發送（忽略已發送記錄）

如果文章已經發送過，系統會提示錯誤。如需重新發送，使用 `--force` 參數：

```bash
npm run newsletter:send -- --slug "2026/01/文章文件名" --force
```

## 方法二：API 調用

### API Endpoint

```
POST https://waynspace.com/api/newsletter
```

### 請求格式

```json
{
  "slug": "2026/01/文章文件名",
  "subject": "自訂標題（可選）",
  "body": "自訂內容（可選）",
  "force": false
}
```

**參數說明**：
- `slug`：文章路徑（必填，如果使用自訂內容則可選）
- `subject`：郵件標題（可選）
- `body`：郵件內容（可選）
- `force`：強制重新發送，即使已發送過（可選，預設為 `false`）

### 使用 curl

```bash
curl -X POST https://waynspace.com/api/newsletter \
  -H "Content-Type: application/json" \
  -d '{
    "slug": "2026/01/旅行的意義-大二寒假環西半部",
    "subject": "新文章發布：旅行的意義"
  }'
```

### 使用 Postman 或類似工具

1. 設定請求方法為 `POST`
2. URL：`https://waynspace.com/api/newsletter`
3. Headers：`Content-Type: application/json`
4. Body（JSON）：
   ```json
   {
     "slug": "2026/01/文章文件名"
   }
   ```

## 郵件格式說明

### 自動生成的郵件格式

當使用 `--slug` 參數時，系統會自動生成郵件內容：

- **標題**：`新文章：{文章標題}`（或使用自訂標題）
- **內容**：
  - 文章標題（H1）
  - 文章摘要（summary 或 excerpt）
  - 閱讀全文連結
  - 文章內容前 500 字預覽

### 如何自訂郵件內容

#### 方法 1：使用命令行參數

```bash
npm run newsletter:send -- \
  --slug "2026/01/文章文件名" \
  --subject "自訂標題" \
  --body "自訂內容（支援 Markdown）"
```

#### 方法 2：使用 API 調用

```json
{
  "slug": "2026/01/文章文件名",
  "subject": "自訂標題",
  "body": "# 自訂標題\n\n這是自訂內容，支援 **Markdown** 格式。"
}
```

**注意**：如果同時提供 `slug` 和 `body`，`body` 會覆蓋自動生成的內容。

## 常見問題

### Q: 如何找到文章 slug？

文章 slug 就是文章的檔案路徑（相對於 `content/` 目錄），格式為：

```
YYYY/MM/文件名（不含 .md 副檔名）
```

**範例**：
- 檔案：`content/2026/01/旅行的意義-大二寒假環西半部.md`
- Slug：`2026/01/旅行的意義-大二寒假環西半部`

### Q: 發送失敗怎麼辦？

1. **檢查環境變數**
   ```bash
   # 確認 BUTTONDOWN_API_KEY 已設定
   echo $BUTTONDOWN_API_KEY
   ```

2. **檢查 API Key 是否有效**
   - 前往 [Buttondown Dashboard](https://buttondown.email/settings/api)
   - 確認 API Key 狀態正常

3. **檢查文章是否存在**
   - 確認文章已經部署上線
   - 在瀏覽器中訪問文章 URL 確認可以正常顯示

4. **查看錯誤訊息**
   - 命令行會顯示詳細錯誤訊息
   - API 調用會返回錯誤狀態碼和訊息

### Q: 如何驗證發送成功？

#### 命令行方式

成功發送後會顯示：
```
✓ 電子報已成功發送！
郵件 ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
✓ 已記錄到追蹤檔案：.newsletter-sent.json
```

#### API 方式

成功時會返回：
```json
{
  "message": "電子報已成功發送",
  "emailId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "slug": "2026/01/文章文件名"
}
```

如果文章已經發送過（且未使用 `force`），會返回：
```json
{
  "error": "此文章已經發送過",
  "sentAt": "2026-01-15T10:30:00.000Z",
  "subject": "新文章：文章標題",
  "emailId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "hint": "如需重新發送，請在請求中加入 \"force\": true"
}
```

### Q: 郵件會發送給誰？

郵件會發送給所有在 Buttondown 中訂閱的用戶。

### Q: 可以測試發送嗎？

可以！在 Buttondown Dashboard 中：
1. 前往 **Subscribers**
2. 添加一個測試用的訂閱者（你自己的郵件）
3. 發送郵件時，測試郵件也會收到

### Q: 如何避免重複發送？

系統會**自動追蹤**已發送的文章，記錄在 `.newsletter-sent.json` 檔案中。

**自動保護機制**：
- 發送前會檢查文章是否已發送過
- 如果已發送，會顯示警告並阻止發送
- 發送成功後會自動記錄到追蹤檔案

**查看已發送記錄**：

```bash
# 查看追蹤檔案
cat .newsletter-sent.json
```

檔案格式：
```json
{
  "articles": [
    {
      "slug": "2026/01/文章文件名",
      "sentAt": "2026-01-15T10:30:00.000Z",
      "subject": "新文章：文章標題",
      "emailId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
    }
  ]
}
```

**強制重新發送**：
如果確實需要重新發送，使用 `--force` 參數（命令行）或 `"force": true`（API）。

## 已發送文章追蹤

系統會自動追蹤已發送的文章，避免重複發送。追蹤資訊儲存在 `.newsletter-sent.json` 檔案中（此檔案已加入 `.gitignore`，不會被提交到 Git）。

### 追蹤檔案位置

```
.newsletter-sent.json
```

### 追蹤內容

每個已發送的文章會記錄：
- `slug`：文章路徑
- `sentAt`：發送時間（ISO 8601 格式）
- `subject`：郵件標題
- `emailId`：Buttondown 郵件 ID

### 查看已發送記錄

```bash
# 查看所有已發送的文章
cat .newsletter-sent.json

# 或使用 jq 格式化顯示（如果已安裝）
cat .newsletter-sent.json | jq
```

### 清除追蹤記錄

如果需要清除所有追蹤記錄（不建議，除非確定要重新發送所有文章）：

```bash
rm .newsletter-sent.json
```
