# 如何撰寫文章

完整指南：從創建文章到上傳圖片

---

## 快速開始

### 1. 創建新文章

在 `content/YYYY/MM/` 下創建 `.md` 文件：

```bash
content/2026/01/我的新文章.md
```

### 2. Frontmatter 格式

```markdown
---
title: 文章標題
date: 2026-01-15T00:00:00.000Z
tags:
  - 標籤1
  - 標籤2
categories:
  - 分類名稱
slug: article-slug
coverImage: 'https://img.waynspace.com/2026/01/article-slug/image.webp'
author:
  name: Wei-Ting Liu
  email: wayntingliu@gmail.com
seo:
  metaDescription: 文章摘要（用於 SEO，150 字以內）
  keywords:
    - 關鍵字1
    - 關鍵字2
summary: >-
  文章摘要（用於卡片顯示，優先使用此欄位）
---
```

**必填欄位：**
- `title` - 文章標題
- `date` - 發布日期（ISO 8601 格式）
- `categories` - 分類列表（第一個為主分類）
- `summary` - 文章摘要（用於卡片顯示）

**可選欄位：**
- `tags` - 標籤列表
- `slug` - URL slug（不填則使用檔名）
- `coverImage` - 封面圖片 URL
- `seo` - SEO 相關設定

## 圖片處理

### 推薦方式：自動處理

1. **將圖片放在文章同目錄下**：
   ```
   content/2026/01/文章-slug/
   ├── 文章-slug.md
   ├── cover.jpg
   └── image1.png
   ```

2. **在文章中使用相對路徑**：
   ```markdown
   ---
   coverImage: "./cover.jpg"
   ---
   
   ![圖片說明](./image1.png)
   ```

3. **執行自動處理腳本**：
   ```bash
   npm run images:process
   ```

**腳本會自動完成：**
- ✅ 轉換圖片為 WebP 格式
- ✅ 壓縮圖片
- ✅ 上傳到 R2 CDN
- ✅ 更新文章中的路徑為 CDN URL
- ✅ 刪除本地圖片

### 圖片 URL 格式

處理後的圖片 URL：
```
https://img.waynspace.com/YYYY/MM/article-slug/image.webp
```

## Markdown 語法

支援標準 Markdown 和 GitHub Flavored Markdown：

- **標題**：`# H1`、`## H2`、`### H3`
- **粗體**：`**粗體**`
- **斜體**：`*斜體*`
- **連結**：`[文字](https://example.com)`
- **圖片**：`![說明](https://img.waynspace.com/...)`
- **列表**：`- 項目` 或 `1. 項目`
- **引用**：`> 引用文字`
- **程式碼**：`` `inline` `` 或程式碼區塊

## 發布流程

1. **撰寫文章** → 在 `content/YYYY/MM/` 創建 `.md` 檔案
2. **處理圖片** → 執行 `npm run images:process`
3. **本地測試** → `npm run dev` 查看效果
4. **提交 Git** → `git add . && git commit -m "..." && git push`
5. **等待部署** → Vercel 自動部署（約 1-2 分鐘）
6. **發送郵件通知** → 執行 `npm run newsletter:send -- --slug "2026/01/文章文件名"`

**詳細說明：** 查看 [如何發送郵件通知](./如何發送郵件通知.md)

## 常見問題

**Q: 圖片顯示不出來？**
- 確認已執行 `npm run images:process`
- 檢查圖片 URL 格式是否正確
- 確認 R2 CDN 已啟用 Public Access

**Q: 文章沒有顯示？**
- 確認 frontmatter 格式正確
- 確認檔案路徑：`content/YYYY/MM/文章.md`
- 重新啟動開發伺服器：`npm run dev`

**Q: summary 欄位如何填寫？**
- 優先使用 `summary` 欄位（用於卡片顯示）
- 如果沒有，會使用 `seo.metaDescription`
- 如果都沒有，會從文章內容自動提取

**Q: 如何修改已發布的文章？**
- 直接編輯 `content/` 下的 `.md` 檔案
- 提交更改後會自動重新部署
